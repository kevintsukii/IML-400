Eloquent JavaScript Chapter 2: The Basics

  • I liked this reading because it laid out the foundation of JavaScript. There are six basic types of values:

    1. Numbers: have 64 bits, allowing for any whole number less than 2^52
    2. Strings: string together a bunch of text characters and are enclosed in quotes, two strings can be concatonated by using a + sign
    3. Booleans: two types: true and false), operators: >,<,=. A piece of code that expresses a value is called an expression.

      4. Larger than expressions are statements, which end with a semicolon. Statements should somehow "change the world," print something to the screen or change the statements that come after. Programs are made of statements that do things.Variables are used to essentially catch and hold values. We always name variables so that you can, for example, run the name in console and retrieve the value. This is starting to feel a lot like middle school algebra.

    4. Objects: are variables too, but can contain many values. They're containers for named values.
    5. Functions: are pieces of program wrapped in a value, often producing a side effect (like an alert window). Every expression that produced a function value can be invoked by puting parathesis after it. Braces {} group statements into blocks.

      6. Loops affect the control flow of a program — they change the order in which statements are executed.

    6. Undefined values: are what you get when you ask for the value of an empty space. Functions which do not return an interesting value such as print and alert also return an undefined value.

Eloquent JavaScript Chapter 3: Functions

  • Functions! They're "canned" code that can go through whenever you need the to, saving time.

  • The variable name is what the resulting function is stored under.
  • Pure functions: are things in themselves. Non-pure functions might return different values based on all kinds of factors and have side effects.
  • Which functions are visible inside a function is determined by the place of that function in the program text—functions defined inside another function retains access to the environment that existed in that function at the point when it was defined.
  • Recursive functions call themselves. They sort of loop, but there is no "while" or "for" or even a local side effect to be seen, though the same effect is produced.
  • Stacks are where the context is stored, "one can visualize this as a stack of contexts. Every time a function is called, the current context is thrown on top of the stack. When a function returns, the context on top is taken off the stack and resumed."

  • I thought this reading was overall useful to understanding functions, but had a lot of information that I didn't find applicable in terms of making functions.
